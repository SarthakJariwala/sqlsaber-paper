Now I can see the structure. The `cards` table has the `power` column, and the `foreign_data` table has the `language` column. They are linked via `uuid`. Let me write and execute the query.


**Executing SQL:**

```sql

SELECT 
    CAST(COUNT(CASE WHEN fd.language = 'French' THEN 1 END) AS REAL) * 100 / COUNT(*) AS percentage_french
FROM cards c
INNER JOIN foreign_data fd 
    ON c.uuid = fd.uuid
WHERE c.power IS NULL OR c.power = '*';

```

### Result

**Approximately 12.98% of cards without power are in French.**

Here's how this was calculated:

1. **Joined** the `cards` table with the `foreign_data` table (which stores translations/language info) using the shared `uuid` key.
2. **Filtered** for cards without power — meaning the `power` field is either `NULL` (no value) or `'*'` (a special placeholder).
3. **Calculated the percentage** by dividing the count of French-language entries among those powerless cards by the total count of all powerless card entries, then multiplying by 100.
