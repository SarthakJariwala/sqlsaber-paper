1. The **`Patient`** table, which holds the admission status to verify if they are followed at the outpatient clinic (`Admission = '-'`).
2. The **`Laboratory`** table, which contains the red blood cell counts (`RBC`) to check for abnormal levels (`<= 3.5` or `>= 6.0`).

We can link these tables together using the patient's ID. Here is the SQL query to find the correct patient IDs:

```sql
SELECT T1.ID
FROM Patient AS T1
JOIN Laboratory AS T2
ON T1.ID = T2.ID
WHERE T1.Admission = '-' AND (T2.RBC <= 3.5 OR T2.RBC >= 6.0);
```

### Explanation:
- `SELECT T1.ID`: This tells the database to only retrieve the patient IDs.
- `FROM Patient AS T1 JOIN Laboratory AS T2 ON T1.ID = T2.ID`: This securely matches a patient's general file with their specific lab test results.
- `WHERE T1.Admission = '-'`: This ensures we are exclusively looking at patients being followed at the outpatient clinic.
- `AND (T2.RBC <= 3.5 OR T2.RBC >= 6.0)`: This filters those lab records to only the ones showing abnormal red blood cell levels based on your context.

*(Note: If a patient had multiple tests showing abnormal RBC levels, their ID will appear in the list multiple times. To get a unique list of patients, you can simply change the first line to `SELECT DISTINCT T1.ID`)*.
